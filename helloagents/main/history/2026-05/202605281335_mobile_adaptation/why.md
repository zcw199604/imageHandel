# 变更提案: 移动端显示与操作适配

## 需求背景
当前 IOPaint 前端主要按桌面端设计，核心编辑区、顶部工具栏、插件入口、底部画笔工具条和侧边设置面板都使用固定位置与桌面交互假设。在移动端验证时出现显示不完整、工具遮挡、触摸绘制/拖拽不稳定等问题，导致用户无法顺畅上传图片、识别水印、调整 mask 和执行修复。

## 产品分析

### 目标用户与场景
- **用户群体:** 使用手机或平板浏览器进行图片水印识别、mask 修正和一键修复的用户。
- **使用场景:** 用户在移动端打开页面，上传图片，选择 Watermark Detector 或手动画笔修正，然后点击修复并保存结果。
- **核心痛点:** 固定桌面布局在窄屏下挤压/溢出；触摸绘制和双指缩放/拖动画布存在冲突；底部工具条和顶部工具栏不适配安全区。

### 价值主张与成功指标
- **价值主张:** 移动端可完成核心闭环：上传图片 → 自动水印检测/手动修正 → 修复 → 查看/保存。
- **成功指标:** 375px 宽度手机视口下无关键控件遮挡或横向溢出；触摸绘制坐标正确；画布可缩放/拖动；水印检测入口可点击；构建通过。

### 人文关怀
移动端布局需考虑单手操作、触控目标大小、刘海屏/底部安全区、误触防护，以及授权图片使用提示仍可见。

## 变更内容
1. 梳理并修复移动端全局布局：页面高度、滚动策略、安全区、header 工具折叠。
2. 调整 Workspace 悬浮工具、插件入口、图片尺寸提示与进度条在窄屏下的位置。
3. 优化 Editor 画布移动端 viewport 计算、触摸绘制、触摸移动、画笔显示与底部工具条布局。
4. 调整 FileSelect、PromptInput、SidePanel/Settings 等固定宽度组件的响应式表现。
5. 增加移动端验证清单和必要自动化检查。
6. 将前端设置项、设置弹层中的用户可见文案替换为简体中文。

## 范围边界
- **范围内:** 前端移动端显示与触控交互适配；设置项用户可见文案中文化；不改后端接口；不改变水印检测算法。
- **范围外:** 原生 App、PWA 离线能力、复杂手势编辑器重写、批量处理移动端 UI。
- **拆分说明:** 本方案包只覆盖移动端可用性闭环，后续可单独规划移动端专属设计系统或 PWA。

## 影响范围
- **模块:** 前端编辑器、Workspace、Header、插件菜单、底部工具条、文件选择、侧边设置面板、设置弹层中文文案、全局样式。
- **文件:** `web_app/src/components/Editor.tsx`、`Workspace.tsx`、`Header.tsx`、`FileSelect.tsx`、`PromptInput.tsx`、`SidePanel/index.tsx`、`Settings.tsx`、`Shortcuts.tsx`（如涉及设置说明）、`web_app/src/globals.css`、`web_app/src/lib/utils.ts`、`web_app/src/hooks/useResolution.tsx`。
- **API:** 无后端 API 变更。
- **数据:** 无数据模型变更。

## 核心场景

### 需求: 移动端基础显示可用
**模块:** 前端布局
用户用手机浏览器打开页面。

#### 场景: 375px 视口打开首页和编辑页
- Header 不横向溢出，核心按钮可触达。
- 上传入口和插件入口在安全区内可见。
- 底部工具条不遮挡系统手势区域，必要时可横向滚动或折行。

### 需求: 移动端图片编辑可用
**模块:** 前端编辑器
用户在手机上上传图片并编辑 mask。

#### 场景: 单指绘制 mask
- 触摸坐标与图片实际位置匹配。
- 绘制过程中页面不被浏览器滚动/缩放打断。
- 画笔大小可调整，运行修复按钮可点击。

#### 场景: 缩放/平移画布
- 用户可通过明确的平移模式或双指手势移动/缩放图片。
- 绘制模式和平移模式有清晰区分，避免误画。

### 需求: 移动端水印检测入口可用
**模块:** 插件菜单
用户在移动端选择 Watermark Detector。

#### 场景: 选择检测策略并生成 mask
- 插件菜单在窄屏下不被裁切。
- CV/OCR、Open Vocabulary、Both 三个选项可点击。
- 生成 mask 后用户可继续手动画笔修正并执行修复。


### 需求: 设置项中文化
**模块:** 设置弹层 / 前端文案
用户打开设置、插件设置、模型设置或快捷键说明时，可以看到简体中文文案。

#### 场景: 移动端查看中文设置项
- 设置项名称、分组标题、按钮、提示说明使用简体中文。
- 中文文案在移动端不挤压关键控件，必要时换行或缩短。
- 技术名词、模型名、参数名可保留英文或中英混排。

## 风险评估
- **风险:** 触摸绘制与画布缩放/平移手势冲突。
- **缓解:** 明确移动端绘制/平移模式；对画布区域应用 `touch-action` 策略；优先保留核心绘制链路。
- **风险:** 桌面端布局被移动端改动破坏。
- **缓解:** 使用响应式 class 和断点，桌面端行为保持不变；执行桌面构建和手工回归。
- **风险:** 侧边面板和弹层在小屏溢出。
- **缓解:** 小屏使用抽屉/全屏弹层或限制最大高度并滚动。
- **风险:** 设置项中文化后文案变长，导致移动端布局进一步拥挤。
- **缓解:** 设置弹层使用响应式换行、短标签和说明文本分层；技术参数保留必要英文缩写。
