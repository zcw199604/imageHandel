# 后端 API

## 目的
提供图像修复、插件执行与模型管理接口。

## 模块概述
- **职责:** FastAPI 路由、插件构建、模型调用、图片编码解码。
- **状态:** 🚧开发中
- **最后更新:** 2026-05-28

## 规范
- 新增可选图像能力优先以插件形式接入。
- 生成 mask 的插件应返回 255 表示需要修复区域、0 表示保留区域。


## 规范补充: 自动水印检测
- `WatermarkDetector` 插件负责生成水印 mask，返回 255 表示需要修复区域，0 表示保留区域。
- 支持 `cv_ocr`、`vl_sam`、`combined` 三种模式；方案2依赖不可用时应降级，不影响服务启动。
- 插件只生成 mask，实际去除继续复用 `/api/v1/inpaint`。

## 规范补充: 水印检测误检防护
- `WatermarkDetector` 默认以主体安全优先：真实开放词汇后端不可用时不得使用泛化显著性算法扩大召回。
- 水印候选需经过连通域形态过滤，避免将人物主体、衣物纹理、自然轮廓等大块或中心区域组件标为待修复 mask。
- 默认 `watermark_dilate` 和 `watermark_max_area_ratio` 应保持保守，膨胀后仍需执行总面积/大块组件安全门禁。
- 半透明浅色文字水印应通过局部背景差分等弱文字线索召回；不同候选线索需先分别过滤再合并，避免与人物轮廓粘连。
- 自动化测试必须覆盖人物/类人体无水印负样本、人物+文字水印正样本、人物身上半透明文字水印、`vl_sam` 无后端降级行为。
