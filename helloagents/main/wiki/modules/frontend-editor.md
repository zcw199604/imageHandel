# 前端编辑器

## 目的
提供图片展示、画笔 mask、插件触发与结果预览交互。

## 模块概述
- **职责:** 编辑状态管理、画笔/临时 mask 管理、插件 API 调用、inpaint 触发。
- **状态:** 🚧开发中
- **最后更新:** 2026-05-28

## 规范
- 自动生成的 mask 应进入 `extraMasks` 或等价临时 mask 流程，允许用户预览和修正。
- 长耗时操作需维护加载状态并显示错误提示。


## 规范补充: 自动水印检测
- 插件菜单提供 Watermark Detector，并允许选择 CV/OCR、Open Vocabulary、Both。
- 自动生成的水印 mask 注入 `extraMasks`，用户可预览、手动画笔修正后再执行修复。
- UI 必须提示仅用于自有或已授权图片。

## 规范补充: 移动端适配与中文设置
- 375×667、390×844、768×1024 视口下，首页上传入口、Header 核心按钮、插件入口、底部画笔工具条和设置弹层不得出现关键控件横向溢出或不可点击。
- 全局页面使用 `100dvh`、安全区变量和 `overscroll-behavior`，编辑画布区域需避免浏览器滚动/缩放打断绘制。
- 移动端编辑器提供明确的绘制/平移模式切换；绘制模式用于单指 mask，平移模式用于移动画布，桌面端保留空格/中键平移行为。
- 触摸与鼠标坐标转换必须基于 canvas 当前显示矩形和真实画布尺寸，`touchend`/`touchcancel` 需正确结束绘制状态。
- 设置弹层、快捷键说明、插件菜单和主要设置项用户可见文案使用简体中文；模型名、插件名和技术枚举可保留英文。
- 授权图片使用提示在移动端仍必须可见，Watermark Detector 菜单继续提示仅用于自有或已授权图片。

## 规范补充: 自动水印 mask 安全预览
- Watermark Detector 生成的 mask 必须先注入可编辑遮罩层，由用户预览确认后再执行修复。
- 默认水印检测参数应偏保守，优先避免人物主体、脸部、身体或衣物被误涂。
- 插件菜单需保留授权图片提示，并提醒用户修复前先预览 mask。
